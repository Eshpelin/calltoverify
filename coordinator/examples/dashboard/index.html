Add a device

A receiver is a spare Android phone (or a Raspberry Pi) with a SIM. Fill in its number, pick the channels it should accept, and connect it — an Android phone scans a QR; a headless Raspberry Pi pastes a one-line command over SSH.

Receiver details

The SIM number is the number end users will contact. You enter it here — it is not read from the SIM.

Connect the receiver

An Android phone scans the QR. A Raspberry Pi has no camera, so you paste a one-line command over SSH instead — both carry the same credentials.

Generate a pairing to connect this receiver.

Integrate

Embed the engine in your Go backend. Your own server becomes the endpoint the receiver talks to — there is no separate service to run.

Go

Mount the device handler and start verifications from your code.

eng, _ := ctv.New(ctx, ctv.Options{ OnVerified: func(ev ctv.Event) { // mark the user verified in your DB }, }) // the receiver app posts here; pair it with a QR from "Add a device" mux.Handle("/ctv/", eng.DeviceHandler("/ctv")) v, _ := eng.StartVerification(ctx, ctv.Params{Channel: "sms"}) // show v.Instructions to the user; poll eng.Status(ctx, v.SessionID)

Other front ends

A drop-in web widget and React / Flutter components render the user-facing flow (text us, missed call, DTMF) against your backend. See the SDKs in the repository.
